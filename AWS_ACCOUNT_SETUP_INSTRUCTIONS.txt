================================================================================

STEP 1: Create AWS Account
--------------------------
1. Go to: https://aws.amazon.com/
2. Click "Create an AWS Account" (top right)
3. Enter email address for the ROOT account (company admin email)
4. Create a strong password
5. Choose account type: "Business"
6. Enter company information:

   - Address: [Your business address]

STEP 2: Add Payment Method (Required for Billing)
-------------------------------------------------
1. After account creation, go to: AWS Console > Billing > Payment Methods
2. Add a credit card or bank account
3. This is required to use any AWS services beyond free tier

STEP 3: Set Up Billing Alerts (Recommended)
-------------------------------------------
1. Go to: AWS Console > Billing > Budgets
2. Click "Create budget"
3. Choose "Cost budget"
4. Set monthly budget amount (recommend: $200 for MVP testing)
5. Set alert threshold at 80% and 100%
6. Add notification email: adonisherome73@gmail.com
7. This prevents surprise charges!

PART 3: ENABLE REQUIRED AWS SERVICES
================================================================================

The developer (Adonis) will need these services enabled:

SERVICE 1: Amazon Bedrock (AI/LLM)
----------------------------------
1. Go to: AWS Console > Amazon Bedrock
2. Region: us-east-1
3. Click "Model access" in left sidebar
4. Click "Manage model access"
5. Enable these models:
   [x] Anthropic - Claude 3.5 Sonnet
   [x] Anthropic - Claude 3 Haiku (backup/cheaper option)
6. Click "Save changes"
7. Wait for access to be granted (usually instant)

SERVICE 2: Amazon Connect (Phone/Voice)
---------------------------------------
1. Go to: AWS Console > Amazon Connect
2. Click "Create instance"
3. Instance name: danny-ai-dental
4. Choose "Store users in Amazon Connect"
5. Create admin user
6. Enable Contact Lens (optional, for analytics)
7. Review and create
8. After creation, claim a phone number:
   - Go to instance > Phone numbers
   - Claim a number (choose US toll-free or DID)
   - Cost: ~$1/month + usage

SERVICE 3: Amazon Transcribe (Speech-to-Text)
---------------------------------------------
1. Already enabled by default
2. No additional setup needed
3. Cost: $0.024 per minute

SERVICE 4: Amazon Polly (Text-to-Speech)
----------------------------------------
1. Already enabled by default
2. No additional setup needed
3. Free tier: 1 million characters/month for first 12 months
4. After free tier: $16 per million characters (neural voices)

SERVICE 5: Amazon DynamoDB (Database)
-------------------------------------
1. Already enabled by default
2. Developer will create tables as needed
3. Cost: On-demand pricing, minimal for MVP

SERVICE 6: Amazon S3 (Storage)
------------------------------
1. Already enabled by default
2. Developer will create buckets for:
   - Call recordings (if consented)
   - Transcripts
   - Logs
3. Enable KMS encryption for HIPAA compliance


================================================================================
PART 4: HIPAA COMPLIANCE SETUP (IMPORTANT)
================================================================================

Since Danny AI handles dental patient information (PHI), AWS must be configured
for HIPAA compliance:

STEP 1: Sign AWS Business Associate Addendum (BAA)
--------------------------------------------------
1. Go to: AWS Console > AWS Artifact
2. Click "Agreements"
3. Find "AWS Business Associate Addendum"
4. Review and accept the agreement
5. This is REQUIRED for HIPAA compliance

STEP 2: Enable CloudTrail (Audit Logging)
-----------------------------------------
1. Go to: AWS Console > CloudTrail
2. Click "Create trail"
3. Name: danny-ai-audit-trail
4. Apply to all regions: Yes
5. Log file SSE-KMS encryption: Enable
6. Create new S3 bucket for logs
7. Enable CloudWatch Logs

STEP 3: Configure KMS (Encryption)
----------------------------------
1. Go to: AWS Console > KMS
2. Create a customer managed key
3. Name: danny-ai-phi-key
4. Use for encrypting:
   - S3 buckets with recordings
   - DynamoDB tables with patient data
